## 主要职责

### 核心设计任务

1. **HarmonyOS NEXT 应用 UI 设计**
   - 页面设计（登录、首页、阅读、学习中心、个人中心等 15+ 页面）
   - 组件设计（按钮、输入框、卡片、弹窗、导航等完整组件库）
   - 交互设计（手势、动画、转场、微交互）
   - 多屏适配（手机 3.5-7.5 英寸、平板 8-13.2 英寸、折叠屏）
   - 遵循 HarmonyOS Design 设计规范（间距、排版、颜色、阴影）

2. **HarmonyOS Design 系统建设**
   - 设计 Token 定义与管理（颜色、排版、间距、圆角、阴影 etc.）
   - Figma 组件库建立与维护（Primary / Components / Variants）
   - 设计规范文档编写（Markdown + 截图 + 交互说明）
   - 与前端工程师 Figma → 代码同步机制建立

3. **阅读体验优化**
   - 阅读页面设计（字体、行距、段距、背景、夜间模式）
   - 书籍列表展示方式优化（网格、列表、大卡片 etc.）
   - 进度条 / 书签 / 笔记 UI 设计
   - 字体选择器设计（大小、字体家族、行距 A/B 测试）

4. **学习功能 UI 设计**
   - 单词本详情页（显示、学习、复习流程）
   - 理解力测试页面（题目展示、作答、反馈、成绩分析）
   - 学习进度追踪可视化（统计图表、成就系统）
   - 推荐内容呈现（个性化书籍推荐、相关阅读）

5. **视觉品牌与市场物料**
   - 应用 Icon 与启动页设计
   - App Store / 华为 AppGallery 商店页面素材（截图、描述配图）
   - 营销物料（社交媒体海报、宣传图、横幅）
   - 品牌色彩与 Logo 延续（Readmigo 海外版 vs 米果智读 国内版）

6. **用户研究与可用性测试**
   - 竞品分析与设计标杆研究
   - 用户画面 / User Journey Mapping
   - 可用性测试（邀请 5-10 个用户测试原型）
   - 设计迭代反馈整理与优化

## 必备要求

### 设计背景

- **工作经验**：3+ 年移动应用 UI/UX 设计，至少 1 款上线的应用从设计到发版全流程经验
- **设计工具**：
  - Figma 高度熟练（组件库、Auto Layout、Prototype、Dev Mode）
  - Sketch 或 Adobe XD（作为备选可用）
- **HarmonyOS 设计**：
  - 学习过 HarmonyOS Design 官方规范（间距系统、字体规范、组件库）
  - 理解与 iOS / Android 设计的差异
  - 推荐有 1+ 个 HarmonyOS 项目或学习案例

### 设计能力

- **信息架构**：能从产品需求推导出页面结构与导航流程
- **交互设计**：理解 Hand-off Spec，能清晰标注交互细节给工程师
- **响应式设计**：适配不同屏幕尺寸的方法论（手机 / 平板 / 折叠屏）
- **设计系统**：理解 Design Token / Component Variants / Spacing Scale 等概念
- **可用性**：熟悉 WCAG 无障碍设计基础（颜色对比度、字体大小、交互热区）

### 思维与行动

- **产品思维**：不只做视觉漂亮，更要理解用户目标与产品指标
- **协作能力**：积极与产品、工程师、市场部门沟通
- **快速迭代**：能在紧张的创业周期中快速输出高质量设计稿
- **文档习惯**：设计稿带详细注释、设计规范完整、更新及时
- **反馈接受能力**：虚心听取反馈，能解释设计决策

## 常见问题

### Q: 是否必须有 HarmonyOS 经验？

**A**: 不是必须。但你需要：
- 快速学习新设计规范的能力
- 对 HarmonyOS Design 规范有真诚学习态度
- 愿意在项目中边做边学

我们会提供 HarmonyOS Design 官方培训与文档支持。

### Q: 股权与薪资的比例？

**A**: 薪资 + 股权组合：
- 全薪资方案：25-40K（推荐大多数人）
- 薪资 + 股权方案：20-32K + 0.3%-0.8%（仅限高度认可者）

### Q: 设计稿如何交付给工程师？

**A**: 
- **Figma Dev Mode**（首选）：工程师直接在 Figma 看代码实现建议
- **设计规范文档**（Markdown）：设计 Token、组件库、交互说明
- **周期性同步**：周一设计评审、周五工程同步（15 分钟）

### Q: 是否支持远程长期？

**A**: 初期 3 个月需定期线下（每月 1-2 周）以与团队建立信任与设计文化。之后，表现优异者可部分远程，但需：
- 每月至少 1 周线下（设计评审、用户测试）
- 参加季度 offsite

### Q: 职业发展路线？

**A**: 
- **设计专家路线**：UI 设计师 → 设计系统负责人 → 设计总监（18-24 个月）
- **产品转向**：设计师 → 产品经理（12+ 个月，转向可行）
- **品牌 / 市场**：设计师 → 品牌运营 / 市场设计主管

### Q: 入职后的学习支持？

**A**: 
- HarmonyOS Design 官方培训（免费报名）
- Figma 高级技巧工作坊（创始人主持）

### Q: 设计工具与订阅？

- Figma Professional（公司支付）

- DevEco Design（华为免费工具）

### Q: 与海外设计团队的协作？

**A**: 
- 初期 80% 精力在国内版本设计
- 20% 支持海外版本（Readmigo）设计同步 / 改进
