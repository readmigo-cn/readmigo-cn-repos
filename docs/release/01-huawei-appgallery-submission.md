## 第一部分：账户与应用注册

### 1.1 开发者账户准备

**前置条件：**
- 华为开发者账户已激活（开发者认证通过）
- 对公账户已绑定（用于提现）
- 企业营业执照副本
- 法人身份证正反面（彩色扫描）

**步骤：**

1. 访问 https://developer.huawei.com/consumer/cn/
2. 点击「应用」→「应用管理」→「创建应用」
3. 选择应用类型：**应用**（非游戏）
4. 填写基本信息：
   - 应用名称：米果智读
   - 默认语言：简体中文
   - 应用分类：**书籍与参考资源**
   - 内容分级：**3+**（默认）
5. 保存后获得应用 ID（AppID）和包名（Package Name）

**注意事项：**
- 包名必须与 HarmonyOS 应用的 bundleName 匹配
- 应用名称审核后不可更改，务必确认

---

### 1.2 应用版本配置

1. 在「版本管理」中点击「新建版本」
2. 配置版本信息：
   - **版本号**: 1.0.0
   - **构建号**: 1 (每次提交递增)
   - **应用大小**: 显示实际 HAP 文件大小
   - **应用分类**: 书籍与参考资源
   - **内容分级**: 3+
3. 点击「下一步」进入文本信息填写

## 第三部分：权限与隐私配置

### 3.1 应用权限声明

在「权限声明」中逐一配置应用使用的权限及场景说明：

| 权限 | 场景说明 | 是否必需 |
|------|---------|---------|
| 网络连接 | 获取图书列表、用户登录、数据同步 | 必需 |
| 访问设备信息 | 应用功能优化、崩溃诊断 | 必需 |
| 存储权限 | 下载书籍到本地、笔记保存 | 必需 |
| 麦克风 | 朗读功能 | 必需 |
| 日历权限 | 阅读计划功能 | 可选 |
| 通讯录 | 邀请好友 | 可选 |

**填写示例：**
```
【网络连接】
应用需要网络连接来获取图书列表、用户账户信息、同步阅读进度。
详见隐私政策：https://readmigo.cn/privacy

【存储权限】
用于本地下载书籍、保存用户笔记和阅读进度。
数据仅存储在设备本地，不上传至服务器。

【麦克风权限】
用于 AI 朗读功能，用户可随时在设置中关闭。
```

### 3.2 隐私政策配置

**隐私政策链接：** https://readmigo.cn/privacy

**政策内容检查清单：**
- [ ] 1. 首段明确列出 **ICP 备案号** （工信部批准）
- [ ] 2. 个人信息收集目的、范围、保留期限
- [ ] 3. 数据存储、传输、销毁方式
- [ ] 4. 第三方分享场景（Sentry、SensorsAnalytics）
- [ ] 5. 用户权利（查询、更新、删除、撤销）
- [ ] 6. 联系方式（support@readmigo.cn）
- [ ] 7. 政策更新说明

**模板（最小化）：**
```
隐私政策

【ICP 备案号】
京 ICP 备 [XXXXXXX] 号

【信息收集】
1. 账户信息：手机号、密码（加密存储）
2. 阅读数据：书籍浏览历史、阅读进度（本地存储）
3. 设备信息：设备型号、系统版本、应用版本（用于优化）
4. 错误日志：App 崩溃情况（通过 Sentry 安全上报）

【数据保护】
- 所有传输使用 HTTPS 加密
- 用户数据加密存储在数据库中
- 离线书籍内容永不上传

【第三方服务】
- Sentry：崩溃诊断（隐私政策：https://sentry.io/privacy/）
- SensorsAnalytics：用户行为分析（隐私政策：https://www.sensorsdata.cn/about/privacy）

【用户权利】
用户可随时通过应用内「设置」→「隐私」查看、修改或删除个人数据。

【联系方式】
隐私问题咨询：support@readmigo.cn
```

## 第五部分：提交审核流程

### 5.1 确认信息提交

1. 登录华为应用市场后台
2. 选择应用 → 版本管理 → 选择版本
3. 检查已填写的信息：
   - ✅ 基本信息（名称、分类、大小）
   - ✅ 截图和图标
   - ✅ 详细描述
   - ✅ 权限声明
   - ✅ 隐私政策链接有效
   - ✅ HAP 文件上传成功

### 5.2 上传 HAP 文件

1. 在「应用文件」模块上传 HAP：
   ```
   entry/build/default/outputs/default/entry-default-release-signed.hap
   ```
2. 系统自动验证：
   - 文件签名
   - 包名匹配
   - 版本号检查
3. 上传成功后显示文件大小和 MD5 值

### 5.3 提交审核

1. 确认所有必填项已完成
2. 点击「提交审核」
3. 阅读《华为应用市场上架协议》
4. 勾选同意并提交
5. 系统显示提交时间和预估审核完成时间（3-5 工作日）

**记录信息：**
```
应用 ID：____________
提交时间：2026-05-01 14:30
预计完成：2026-05-06 （3 工作日）
审核单号：____________
```

## 第六部分：审核阶段监控

### 6.1 审核状态监控

**日常检查：**
- 每天 2 次登录后台检查审核进度
- 状态包括：
  - 待审核（灰色）
  - 审核中（蓝色）
  - 已通过（绿色）
  - 已拒绝（红色）

**沟通渠道：**
- 华为审核意见通常通过应用后台「消息」发送
- 关注邮箱（开发者注册邮箱）

### 6.2 常见拒绝原因与处理

**发现拒绝后的流程：**

1. **立即记录** → 截屏保存拒绝原因
2. **分析根因** → 对照「应对工具包」(03-rejection-response-toolkit.md)
3. **修复代码 / 文案** → 详见工具包对应章节
4. **本地测试** → 验证修复生效
5. **重新提交** → 上传新 HAP 或更新文案
6. **标注变更记录** → 在「开发者回复」中说明修改内容

**拒绝原因示例：**

| 原因 | 示例 | 应对 |
|------|------|------|
| 隐私政策不完整 | ICP 备案号缺失 | 添加备案号，提交新版政策链接 |
| 应用功能描述不准确 | 声称支持功能但未实现 | 更新应用描述文案，去除虚假宣传 |
| 权限过度 | 申请麦克风但无朗读 | 移除不必要权限 |
| 内购缺失 | 宣传订阅功能但无法购买 | 完成内购配置，测试购买流程 |
| 应用崩溃 | 某机型无法启动 | 修复 Bug，上传新 HAP |

## 第九部分：加急审核申请

### 9.1 适用场景

加急审核适用于以下情况（非常见）：

1. **关键 Bug 修复** → 上个版本有严重缺陷
2. **安全漏洞补丁** → 发现隐私或数据泄露风险
3. **内容合规更新** → 政策调整需立即更新

### 9.2 申请流程

1. 在应用后台进入「消息」或「工作单」
2. 选择「提交加急申请」
3. 填写表单：
   - 原因类型：选择 Bug 修复 / 安全 / 合规
   - 详细说明：300 字以内，详述 bug 的严重程度和影响用户数
   - 修复内容：说明新版本的改进点
4. 提交审核

**成功概率：** 30-50%（取决于原因严重程度）
**加急审核周期：** 12-24 小时
