# 米果智读内容获取策略（22 周产品路线图）

## 执行摘要

内容库规模是本项目成败的关键因素。当前最大风险：**W1-W22 无法积累足够高质量的英语学习内容**。

本文档制定三阶段、三渠道并行的获取战略，通过公版书、出版社合作、社区协作三线突进，目标在产品发布前积累 1000+ 本优质书籍，为用户提供充分的选择与学习连贯性。

**预算估算**：¥500,000 - ¥1,000,000（W1-W22 全周期）

---

## 内容目标与里程碑

### 分阶段目标

| 阶段 | 时间 | 目标书籍数 | 质量标准 | 上线目标 |
|------|------|----------|---------|---------|
| **Phase 1 - MVP** | W1-W8 | 500-800 本 | 公版 + 精选教材 | 产品 Beta 发布 |
| **Phase 2 - 扩展** | W9-W16 | 3,000-5,000 本 | 多类型、多难度 | 产品正式发布 |
| **Phase 3 - 优化** | W17-W22 | 10,000+ 本 | 完整生态，用户反馈迭代 | 融资前达成 |

### 内容特征

**目标受众**：英语学习者（ESL / EFL），从 A1（初级）到 C1（高级）

**优先级**：
1. **英文原著**（占 60-70%）：Project Gutenberg 精选、英文教材、现代英文读物
2. **中英双语**（占 20-25%）：对标、分级、促进理解
3. **其他语言**（占 5-10%）：俄语、西班牙语、法语等支持语言

**难度分级**：遵循 CEFR 框架（A1 → A2 → B1 → B2 → C1）

## 三大内容获取渠道

### 渠道 A：公版书（Public Domain）

**概况**：著作权已过期的书籍，全球免费使用，零版权成本。

**优势**：
- 成本极低（仅需数字化、排版、审核）
- 法律风险最小（已确认公版）
- 质量参差，但精选可得优质内容

**劣势**：
- 内容多为 100+ 年前著作（部分过时但经典）
- 数字化质量需人工审核（OCR 错误、排版混乱）
- 覆盖面有限（无法满足 100% 内容需求）

#### 数据源及时间表

| 数据源 | 特点 | 计划采集 | 时间表 |
|--------|------|--------|--------|
| **Project Gutenberg** (70K+ 本) | 全球最大公版库，英文占 60%，自动化下载可行 | 1,500-2,000 本（精选） | W1-W4 |
| **Standard Ebooks** (高质量排版) | 公版精排版，美感极佳，更新缓慢 | 500-800 本 | W1-W4 |
| **Internet Archive / Open Library** | 扫描 + OCR，内容庞大但质量参差 | 500-1,000 本（筛选） | W2-W6 |
| **中国国学经典 / 公版** | 鲁迅、老舍、巴金等中文作品（若有英译版） | 100-200 本 | W3-W8 |

**执行流程**：
```
W1: 与 Project Gutenberg 团队协调批量接入权限 (if needed)
W2: 自动爬虫 + 数据处理（转换格式、去重、基础清洗）
W3-W4: 人工审核（排版、OCR 质量、内容完整性）
W4: 导入生产库，建立版本追踪
```

**风险管理**：
- **版权确认**：虽为公版，仍需确认每本书在中国/美国的版权状态
- **内容审核**：政治敏感（如涉及苏联时期著作）、宗教内容、不雅用语检查
- **OCR 质量**：部分扫描版质量差，需文本审核或重新排版

**成本估算**：
- 自动化数据处理：¥30,000-50,000
- 人工审核（50 本 × 2 小时 × 200 元/小时）：¥20,000
- 排版与修正：¥50,000-100,000
- **小计**：¥100,000-150,000

### 渠道 B：教材出版社合作（商业内容）

**概况**：与国内教材出版社签约，获得分级读物与教学用书的数字版权。

**优势**：
- 内容高质，专为学习者设计
- 难度分级明确，与课程体系对应
- 出版社有发行资源（学校渠道 + 营销支持）
- 品牌合作机会（联合营销）

**劣势**：
- 需要商务谈判（周期 4-8 周）
- 分成模式复杂（按销售量、用户数或固定费用）
- 部分内容可能有其他平台独占权限

#### 主要合作对象

| 出版社 | 重点内容 | 难度覆盖 | 合作方式优先级 | 初步联系方式 |
|--------|--------|--------|--------------|------------|
| **外研社**（外语教学与研究出版社） | 牛津双语经典（Oxford Bookworms）、新概念英语 | A1-B2 | 1. 分成（70/30） 2. 买断 3. API 接入 | 官网 + 北京总社：010-88819000 |
| **北外出版社** | 英语分级读物、语言学习配套 | A1-B2 | 同外研社 | 北京外国语大学出版社：010-88819098 |
| **上外出版社** | CEFR 分级读物、原版引进 | A1-C1 | 同上 | Shanghai: 021-35010700 |
| **商务印书馆** | 经典著作双语版、词典 | B1-C1 | 1. 特定书单采购 2. 版权许可 | 北京 400-669-0408 |
| **华东师大出版社** | 教育类、儿童分级读物 | A1-B2 | 1. 教材采购 2. 内容授权 | 上海 021-54301298 |

#### 合作谈判关键点

**洽谈目标**：
1. 获得 100-200 本精选出版社内容的数字版权
2. 优先级：分成模式 > 买断 > API 接入（按灵活性排序）

**关键议题**：

| 议题 | 建议方案 | 备注 |
|------|--------|------|
| **价格模式** | 分成 70/30（Readmigo 70%）或按用户数计费 | 初期分成优于买断（现金压力小） |
| **独占权限** | 仅限中国区非母语教学市场 | 明确海外版本不受限制 |
| **内容更新** | 每年 1-2 次版本更新，Readmigo 维护排版 | 避免出版社频繁介入 |
| **推广支持** | Readmigo 承诺在 App 内突出展示 + 联合营销 | 出版社愿意的话可获校园渠道 |
| **合规** | Readmigo 负责网信办备案、内容审核 | 出版社提供内容证明（版权清单） |

**合同期限**：3-5 年，产品上线后可根据销售情况续约

### 渠道 C：社区与海外资源共享

**概况**：与国内外学习社区、开源组织合作，获得免费或低成本内容。

**优势**：
- 成本接近零（志愿者驱动）
- 社区参与度高，口碑传播强
- 可获得用户创作内容（如笔记、精讲）

**劣势**：
- 内容质量不均一
- 版权归属可能复杂（社区内容多无明确授权）
- 需要社区管理与审核工作量大

#### 合作渠道

| 平台 / 组织 | 内容类型 | 合作方式 | 时间表 |
|------------|--------|--------|--------|
| **Standard Ebooks 中文社区** | 公版精排版翻译 | 内容共享协议 + 社区致谢 | W2-W4 |
| **豆瓣阅读 / 简书** | UGC 用户创作内容、学习笔记 | 内容聚合 + 分利（如有） | W6-W10 |
| **开源教育项目** （如 MIT OpenCourseWare 中文版）| 学习材料、讲义 | 署名使用 | W4-W8 |
| **英语学习社区**（如掘金、V2EX、Reddit r/English Learning） | 用户推荐书单、讨论精华 | 内容采集 + 社区透明度 | 持续 |

**执行**：
```
W2-W3: 发起"Readmigo 内容建设者"计划，邀请社区贡献内容
       - 众包 OCR 修正、双语翻译、笔记创作
       - 激励：App 内货币、排行榜、认可标签

W4-W8: 建立"Readmigo 特约编辑"角色
       - 邀请高质量贡献者成为编辑
       - 负责书单推荐、质量审核
       - 月度稿费 ¥500-2000

W9+:   持续运营，建立UGC生态
```

**成本估算**：
- 社区运营人员（1人，50% 精力）：¥10,000-15,000/月 × 6 月 = ¥60,000-90,000
- 贡献者激励：¥50,000-100,000
- **小计**：¥110,000-190,000

## 四：备选方案与风险应对

### 如果上述渠道延期或失败

#### 备选方案 1：自有原创内容

**时间表**：如 W6 未取得出版社进展，启动此方案

**执行**：
- 与英语教师、内容创作者合作，创作分级读物
- 每周 1-2 篇 1000-3000 字的原创内容
- 成本：¥500-2000/篇 × 100 篇 = ¥50,000-200,000（W8-W16）

**局限**：输出速度慢，难以在 W8 前形成规模内容库

#### 备选方案 2：完全聚焦公版内容

**调整**：
- 放弃商业内容合作，完全依赖公版书
- 精选 Project Gutenberg 中"可靠"的 5,000-10,000 本公版书
- 投入更多审核与排版成本

**优点**：法律风险极低，成本可控
**缺点**：内容陈旧，难以吸引现代学习者，产品差异化低

#### 备选方案 3：平台合作与内容聚合

**模式**：不自己持有内容，而是与书籍平台 API 对接
- 与喜马拉雅、得到、樊登读书等 UGC 库对接
- 用户在 Readmigo 搜索后链接到第三方平台

**缺点**：无法实现"离线阅读"、"自定义高亮"等核心功能

## 内容审核与合规流程

**网信办备案要求**：文化内容、儿童内容、出版物需分类标签与审核

### 审核流程 SOP

```
1. 自动审核（W1 上线即启动）
   - 敏感词扫描（政治、暴力、色情、毒品、宗教煽动）
   - 使用百度内容审核 API 或讯飞 API（成本 ¥0.05-0.15/次）
   - 标记高风险内容为"待人工审核"

2. 人工复审（内容专家）
   - 教育内容专家（英语教师）核对语言适当性
   - 出版社编辑初审确保质量
   - 法务审查（确保无版权侵害）
   - 频率：每周批量审核（2-3 次）

3. 分类与标签
   - CEFR 难度标记（A1-C1）
   - 内容类型（小说、教材、传记、诗歌等）
   - 适龄标记（All Ages / 13+ / 16+ / 18+）
   - 特殊标记（含暴力、敏感政治话题等）

4. 抽样监控
   - 每周抽检已上线的 50 本书籍
   - 用户举报响应（2 小时内初审，24 小时内决策）
   - 月度审核报告提交网信办（合规日志）
```

**成本估算**：
- 百度/讯飞 API 调用：1000 本 × ¥0.1 = ¥100
- 人工复审（1 人，50% 精力）：¥15,000/月 × 12 月 = ¥180,000
- 法务咨询：¥50,000-100,000
- **小计**：¥230,000-280,000

## 技术与数据库基础建设

### 内容库架构

**数据表**：
```sql
-- 书籍表
CREATE TABLE books (
  id UUID PRIMARY KEY,
  title VARCHAR(255) NOT NULL,
  author VARCHAR(255),
  cover_image_url TEXT,
  description TEXT,
  source TEXT, -- 'public_domain' / 'publisher' / 'community'
  cefr_level VARCHAR(2), -- A1, A2, B1, B2, C1, C2
  content_type VARCHAR(50), -- 'fiction' / 'non-fiction' / 'textbook' / 'poetry'
  word_count INT,
  estimated_reading_hours FLOAT,
  language VARCHAR(5) DEFAULT 'en',
  has_chinese_translation BOOLEAN DEFAULT FALSE,
  reviewed_at TIMESTAMP,
  published_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT NOW()
);

-- 书籍内容表
CREATE TABLE book_chapters (
  id UUID PRIMARY KEY,
  book_id UUID REFERENCES books(id),
  chapter_index INT,
  title VARCHAR(255),
  content TEXT, -- 实际文本内容
  word_count INT,
  created_at TIMESTAMP
);
```

**CDN 存储**：
- 书籍图片：华为云 OBS（中国节点）
- 书籍文本：PostgreSQL（Neon 或华为云 GaussDB）
- 缓存：Redis 热门书籍元数据

### 内容导入工具链

**W1-W2 开发的工具**：
```
1. OCR 与文本清洗脚本（Python + Tesseract / Paddle OCR）
2. 格式转换器（EPUB/MOBI/PDF → Markdown/JSON）
3. 版权检查脚本（自动对比 copyright database）
4. 批量导入工具（CSV / API 接口，支持幂等操作）
```

**成本**：¥20,000-50,000（开发）

## 关键依赖与风险管理

### 硬依赖（Blocker）

| 风险 | 影响 | 缓解方案 | 优先级 |
|------|------|--------|--------|
| **主体公司未注册** | 无法签署出版社协议、网信办备案 | W1 优先完成工商注册与税号申请 | 🔴 高 |
| **网信办内容审核备案** | 无法上线任何非公版内容 | W2 启动备案流程，预留 4-6 周审核期 | 🔴 高 |
| **出版社谈判失败** | Phase 2 内容库无法扩展到 3000+ 本 | 启动"自有原创"与"社区众包"备选方案 | 🟠 中 |
| **OCR 质量不达标** | 公版书大量错误，审核工作量翻倍 | 采购专业 OCR 服务（汉字识别）或手工输入 | 🟠 中 |

### 软风险（提前规避）

| 风险 | 表现 | 规避方案 |
|------|------|---------|
| **内容过于陈旧** | 用户反馈"全是 19 世纪小说" | 出版社合作占 30-40%，引入现代内容 |
| **难度分级错误** | 初级用户收到高级书籍，用户流失 | 邀请英语教师审核难度标签，A/B 测试调整 |
| **版权纠纷** | 后期发现某本书版权有争议，需下架 | 与法务建立"版权检查清单"，签约前逐一确认 |
| **社区管理失控** | UGC 内容包含违规内容，平台受罚 | 建立严格的审核规范与举报机制 |

**关键成功因素**：

  - **应对**：W1 即启动，咨询专业备案机构加快流程
