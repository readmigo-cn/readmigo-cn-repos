## 二、算法用途与适用场景

### 2.1 算法类型分类

米果智读中使用的推荐算法可分为以下几类：

| 算法用途 | 触发方式 | 算法类型 | 个性化程度 |
|---------|--------|--------|----------|
| **书籍推荐** | 用户主动浏览书架 | 协同过滤 + 内容匹配 | 高 |
| **章节推荐** | 用户完成一本书后 | 难度匹配 + 主题推荐 | 中 |
| **学习计划** | 用户首次注册或手动请求 | AI 生成个性化计划 | 高 |
| **词汇复习排序** | 用户查看生词本 | 艾宾浩斯遗忘曲线 | 中 |
| **AI 功能触发** | 用户长按单词或主动查询 | 用户主动触发，非推荐 | 无 |

### 2.2 推荐算法详细说明

#### 算法 1：书籍推荐

**场景**：用户打开"推荐"标签页，系统推荐符合其水平的英文公版书

**算法机制**：

```
[用户画像构建]
  ├─ 已读书籍列表
  ├─ 阅读完成度
  ├─ 标注/笔记数量
  ├─ 评分记录
  └─ 阅读时间分析 → 阅读水平推断（初级/中级/高级）

[书籍特征提取]
  ├─ 词汇难度（Flesch Reading Ease 指数）
  ├─ 书籍长度（字数）
  ├─ 主题标签（冒险/爱情/科幻/古典等）
  ├─ 评分（其他用户评分聚合）
  └─ 流行度（阅读人数）

[推荐计算]
  ├─ 协同过滤：找相似用户喜欢的书
  ├─ 内容匹配：根据用户兴趣和难度匹配书籍
  ├─ 多样性：确保推荐结果不重复
  └─ 排序：按匹配度排序，返回 Top 10

[推荐展示]
  ├─ 显示书名、作者、难度标签
  ├─ 显示"为什么推荐你"（透明化解释）
  └─ 用户可点击调整难度/主题偏好
```

**算法特点**：
- ✅ **可解释性强**：用户可看到推荐理由
- ✅ **可调整**：用户可改变偏好重新推荐
- ❌ **不使用社交信息**：不基于用户的朋友或粉丝
- ❌ **不使用人口统计**：不考虑年龄、性别、地区
- ❌ **不形成信息茧房**：主动推荐不同主题的书

**透明化解释示例**：
```
推荐理由：
✓ 你最近读完《Pride and Prejudice》，难度评分 7.5/10
✓ 这本书难度 7.2/10，与你的水平匹配
✓ 同样喜欢19世纪文学的用户中，有80%评价这本书为5星
✓ 你的阅读标签中有"浪漫文学"，这本书也属于此类

[不感兴趣] [调整推荐]
```

#### 算法 2：学习计划推荐

**场景**：用户首次注册或点击"生成个性化学习计划"

**算法机制**：

```
[用户信息收集]
  ├─ 问卷：英语水平（初级/中级/高级）、学习目标、可用时间
  ├─ 行为数据：注册后的首批阅读选择
  └─ 性能数据：首本书的完成度、笔记量

[推荐计划生成]
  ├─ AI（基于 LLM）根据用户等级生成：
  │  ├─ 8 周学习计划（每周 5 本书）
  │  ├─ 难度递进曲线（Week 1: 5.0→Week 8: 8.0）
  │  └─ 主题多样性混搭
  │
  ├─ 人类可读输出：
  │  └─ "根据你的初级英语水平，建议从…开始"
  │
  └─ 用户同意 → 计划入库
```

**计划调整机制**：
- 用户每周可调整下周计划
- 如连续两周未完成 → 自动降低难度
- 如连续两周超额完成 → 自动提高难度
- 用户可手动"重新生成计划"

**算法特点**：
- ✅ **动态自适应**：根据用户进度实时调整
- ✅ **透明化**：用户看到调整逻辑
- ✅ **可干预**：用户可手动调整
- ❌ **不强制推送**：计划仅作为建议，不强制执行

#### 算法 3：词汇复习排序

**场景**：用户打开"生词本"或"复习计划"

**算法机制**：

```
[艾宾浩斯遗忘曲线应用]
  用户学习词汇 → 记录学习时间 → 根据遗忘曲线安排复习时间

[复习周期计算]
  ├─ 第 1 次学习：5 分钟后复习
  ├─ 第 2 次复习：24 小时后复习
  ├─ 第 3 次复习：3 天后复习
  ├─ 第 4 次复习：7 天后复习
  ├─ 第 5 次复习：15 天后复习
  └─ 之后：每月复习一次

[复习优先级排序]
  ├─ 优先级 1（紧迫）：应该在 1-3 小时内复习的词汇
  ├─ 优先级 2（近期）：应该在 1-7 天内复习的词汇
  ├─ 优先级 3（后续）：应该在 8-30 天内复习的词汇
  └─ 按优先级顺序展示给用户
```

**算法特点**：
- ✅ **基于科学规律**：艾宾浩斯遗忘曲线经过验证
- ✅ **完全可预测**：用户可看到下次复习时间
- ✅ **无个性化偏差**：同一词汇的复习周期对所有用户相同
- ❌ **不涉及用户画像**：不记录用户背景信息

## 三、推荐算法的限制与保护

### 3.1 明确不使用的算法机制

| 算法类型 | 说明 | 米果智读是否使用 |
|---------|------|----------------|
| **用户社交网络分析** | 基于用户关系推荐 | ❌ 不使用 |
| **用户人口统计学分析** | 基于年龄、性别、地区推荐 | ❌ 不使用 |
| **用户点击行为微观分析** | 追踪用户每个点击进行微调 | ⚠️ 部分使用（用于改进算法，已脱敏） |
| **信息茧房强化** | 刻意强化用户已有偏好 | ❌ 不使用 |
| **内容操控** | 刻意操控用户决策 | ❌ 不使用 |
| **成瘾性设计** | 无限滚动、消息轰炸等 | ❌ 不使用 |

### 3.2 推荐结果的多样性保证

**多样性目标**：确保推荐内容涵盖不同主题和风格

**实现机制**：

```
[Top 50 候选书籍池生成]
  ├─ 协同过滤排名：Top 20
  ├─ 内容匹配排名：Top 20
  ├─ 热门书籍：Top 5
  ├─ 新入库书籍：Top 5
  └─ 合并去重 → 50 本候选

[多样性筛选]
  ├─ 从 50 本中选择 10 本
  ├─ 确保至少 3 个不同的主题标签
  ├─ 难度覆盖：简单、中等、困难各占 30%、40%、30%
  ├─ 来源多样：不只推荐热门书籍
  └─ 最终排序：按匹配度 + 多样性综合排序

[推荐展示]
  ├─ 第 1-3 本：最匹配用户兴趣的书
  ├─ 第 4-7 本：多样化推荐（不同主题）
  └─ 第 8-10 本：鼓励探索（新颖/小众书籍）
```

**多样性指标**：
- 推荐结果中主题标签数 ≥ 5
- 推荐结果中难度分布：简中困 = 30%:40%:30%
- 推荐结果中热门度分布：热门 50% + 小众 50%

### 3.3 用户偏好的自主控制

**用户控制面板**位置：设置 → 推荐与隐私

#### 控制 1：开关推荐算法
```
[推荐功能] 
  ├─ 开启：展示个性化推荐
  └─ 关闭：仅展示热门书籍（无个性化）
```

#### 控制 2：调整推荐偏好
```
[推荐偏好设置]
  ├─ 难度范围：初级 / 中级 / 高级（多选）
  ├─ 主题兴趣：选择感兴趣的主题（冒险、爱情、科幻等）
  ├─ 更新频率：每天 / 每周 / 不推荐
  └─ 推荐多样性：倾向于匹配 ← → 倾向于探索
```

#### 控制 3：重置算法画像
```
[重置学习画像]
  点击后：
  ├─ 清除系统对用户阅读偏好的理解
  ├─ 清除推荐历史
  └─ 下次打开推荐页面时从零开始学习
```

#### 控制 4：查询与删除
```
[查询我的数据]
  ├─ 查看系统记录的用户画像（阅读水平、主题兴趣等）

  └─ 查看算法特征（系统认为用户关心的特征）

[删除我的推荐数据]
  ├─ 删除推荐历史
  ├─ 删除用户画像
  └─ 立即生效，30 天后物理删除
```

## 四、推荐算法的风险评估

### 4.1 偏见与歧视风险

**风险**：推荐算法可能因数据偏见导致某些用户群体被歧视

**米果智读的缓解措施**：

| 风险类型 | 具体风险 | 缓解措施 | 评估 |
|---------|--------|--------|------|
| **性别偏见** | 向女性推荐爱情小说，向男性推荐科幻小说 | 不使用性别数据；主题推荐与性别无关 | ✅ 低风险 |
| **年龄偏见** | 向年轻人推荐简单书，向老年人推荐复杂书 | 不使用年龄数据；难度仅基于阅读表现 | ✅ 低风险 |
| **地域偏见** | 区别对待不同地区用户 | 不收集地域信息；推荐逻辑与地域无关 | ✅ 低风险 |
| **教育程度偏见** | 因教育背景推荐不同难度 | 不使用教育背景数据；仅看阅读行为 | ✅ 低风险 |

### 4.2 信息茧房风险

**风险**：算法可能强化用户已有偏好，导致视野狭窄

**米果智读的缓解措施**：

```
[主动打破茧房]
  ├─ 推荐结果中至少 30% 是用户未曾涉及的主题
  ├─ 定期推荐小众/冷门书籍（曝光小众内容）
  └─ 提供"随机推荐"功能（完全随机选择）

[透明化告知]
  ├─ 推荐页面顶部：说明"我们有意推荐多样化内容"
  ├─ 推荐结果旁：标记"新主题"或"小众精选"
  └─ 推荐理由中说明："…为了拓宽你的阅读视野"

[用户自主控制]
  └─ 用户可在设置中调整"多样性偏好"滑块
```

### 4.3 不当推荐风险

**风险**：算法可能推荐含有不当内容的书籍

**米果智读的缓解措施**：

```
[源头控制]
  └─ 所有推荐的书籍来自经过审核的白名单
     （见《内容审核备案》）

[推荐前二次审核]
  ├─ 书籍入库时已审核过一次
  ├─ 加入推荐前再审核一次（确保未被修改）
  └─ 不推荐被用户举报或被监管部门标记的书

[推荐后监督]
  ├─ 用户可举报"不当推荐"
  ├─ 合规团队 24 小时内调查
  └─ 确认不当 → 下架该书、优化算法
```

## 六、算法变更与监督

### 6.1 重大变更通知

如果推荐算法发生重大变更（如更换算法模型、改变推荐逻辑等），米果智读将：

1. **提前通知用户**（变更前 7 天）
   - APP 推送通知
   - 邮件通知（已订阅用户）
   - 网站公告

2. **说明变更内容**
   ```
   我们更新了推荐算法：
   - 新增"难度自适应"功能
   - 改进了"多样性保证"机制
   - 提高了推荐速度
   
   这对你有什么影响：
   ✓ 推荐会更符合你的水平
   ✓ 你会看到更多不同风格的书
   ✗ 需要重新学习你的偏好（3 天内恢复）
   ```

3. **提供退出选项**
   ```
   如果你不满意新版本，可以：
   [切换回旧版本] （7 天内可随时切换）
   [关闭个性化推荐]（使用通用推荐列表）
   ```

### 6.2 内部审计

**频率**：每季度一次

**审计内容**：
- 推荐算法的准确度（用户是否满意推荐）
- 偏见检测（是否存在性别/年龄偏见）
- 多样性评估（推荐结果是否足够多样）
- 风险识别（是否存在新的风险）

**审计流程**：
```
[采样用户]
  → 随机选择 100-500 个活跃用户
  
[数据分析]
  ├─ 推荐转化率：点击推荐书籍的用户比例
  ├─ 满意度：评价为 5 星的推荐比例
  ├─ 多样性指数：推荐结果的主题覆盖范围
  └─ 偏见指数：不同用户群体的推荐差异
  
[生成报告]
  └─ 识别问题、提出改进建议
  
[整改措施]
  └─ 实施改进，下次审计验证
```
