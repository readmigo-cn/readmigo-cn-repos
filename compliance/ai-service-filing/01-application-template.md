## 3. 算法机制机理

### 3.1 模型来源
- 主模型：DeepSeek V3.2（深度求索 / 已完成网信办备案）
- 备选：通义千问、智谱 GLM、文心一言（均已备案）
- ✅ 不使用海外模型，零数据出境

### 3.2 应用场景
- **划词查询**：用户长按英文单词，模型返回中英解释
- **句子翻译**：英→中、中→英
- **生词解释**：基于上下文的个性化解释
- **图书摘要**：（受限场景，仅 Pro 用户）
- **内容问答**：基于当前章节的封闭式问答

### 3.3 输出限制
- 禁止生成 / 输出涉政、暴力、色情、赌博内容
- 通过 prompt 工程 + 关键词过滤 + DeepSeek 自带内容安全双重把关
- 用户可见的免责声明：AI 生成内容仅供学习参考

### 3.4 数据来源
- **训练数据**：本服务**不进行模型训练**，仅消费 API
- **图书内容**：Standard Ebooks（公版）+ Project Gutenberg（公版）+ 中文公版（书格 / Wikisource / CTEXT）
- 所有图书内容上线前经过：
  - 自动化关键词审核（百度内容审核 API）
  - 人工抽查（每批次 5%）
  - 黑名单机制

## 4. 安全评估

### 4.1 内容安全
- 输入侧：用户 prompt 关键词过滤
- 输出侧：DeepSeek 内置内容安全 + 二次审核
- 异常监控：日志全量留存 6 个月
- 应急预案：发现违规立即下架内容并上报

### 4.2 数据安全
- 用户输入仅用于即时响应，不留存训练数据
- 个人信息加密传输（HTTPS）+ 加密存储（华为云 KMS）
- 数据本地化：全量数据在华为云北京四（cn-north-4）

### 4.3 个人信息保护
- 收集最小化原则
- 用户可随时删除账号
- 14 岁以下需监护人同意（参见《未成年人保护协议》）
- 完整《隐私政策》参见 `compliance/policies/data-security-policy.md`
