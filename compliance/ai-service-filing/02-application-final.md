### 3.3 输出安全过滤机制

#### 一级过滤：Prompt 设计（预防性）

系统 Prompt 明确约束 LLM 输出范围：

```
你是米果智读的 AI 英语学习助手。

**核心约束**：
1. 仅提供英语学习相关的内容解释
2. 严禁生成：
   - 政治敏感内容（反宪法、反国家、分裂国家等）
   - 暴力、恐怖主义内容
   - 色情、淫秽内容
   - 宗教极端主义内容
   - 宣扬违法犯罪的内容
3. 对于包含上述敏感内容的用户查询，直接拒绝并说明原因
4. 输出必须符合《个人信息保护法》，不泄露他人隐私
5. 涉及未成年人时，严禁成人内容

**语境**：用户在学习以下书籍：[书籍名称]
当前章节：[章节内容摘要]

用户的查询是：[用户查询]

请基于以上信息提供学习帮助。
```

#### 二级过滤：内容安全检测（事中）

**方案 A：百度 AI 内容审核 API**

调用百度内容安全检测 API 检测 LLM 生成内容，覆盖以下维度：

| 审核维度 | 检测标签 | 阈值 | 处理 |
|--------|--------|------|------|
| **政治敏感** | 反宪法、反政府、分裂国家等 | 置信度 > 80% | 拒绝返回 |
| **色情低俗** | 淫秽、色情、性暗示 | 置信度 > 85% | 拒绝返回 |
| **暴力恐怖** | 暴力、血腥、恐怖主义、犯罪方法 | 置信度 > 80% | 拒绝返回 |
| **宗教极端** | 邪教、宗教极端主义 | 置信度 > 75% | 拒绝返回 |
| **谣言骗诈** | 散布谣言、诈骗信息 | 置信度 > 75% | 拒绝返回 |
| **涉未成年** | 未成年人保护、儿童安全 | 置信度 > 90% | 拒绝返回 |
| **侵权侮辱** | 侮辱他人、诽谤、侵犯隐私 | 置信度 > 75% | 拒绝返回 |

**方案 B：自研敏感词库**

基于国家相关法律法规，自主维护敏感词库，包含：
- 政治敏感词汇（5,000+ 条）
- 涉黑社会用语（2,000+ 条）
- 色情低俗词汇（3,000+ 条）
- 暴力恐怖词汇（1,500+ 条）
- 宗教极端词汇（800+ 条）
- 品牌与个人隐私（动态维护）

**双重检测机制**：
- 内容同时经过百度 API 和自研词库检测
- 任一检测不通过 → 内容被拦截
- 拦截记录日志 → 用于持续优化

#### 三级过滤：人工复审（事后）

**用户举报机制**：
- APP 内提供"反馈/举报"按钮
- 用户可标记"不当内容"
- 合规团队 24 小时内人工复核
- 确认违规内容 → 立即下架并处理相关记录

### 3.4 不同场景的审核差异

#### 场景 1：划词查询
- **输入**：单个英文单词或短语
- **输出**：中英释义、发音、例句
- **审核强度**：低（词汇本身难以违规）
- **风险**：极低

#### 场景 2：句子翻译
- **输入**：英文句子（来自当前书籍）
- **输出**：中文翻译
- **审核强度**：中（需检测翻译是否歪曲原意）
- **风险**：低到中

#### 场景 3：内容问答
- **输入**：用户自定义问题
- **输出**：基于书籍内容的回答
- **审核强度**：高（可能偏离源内容）
- **风险**：中到高
- **额外措施**：限制回答范围在 ±500 字符内源内容周围

- 协议：HTTPS/TLS 1.3

**检验方式**：
- 所有 HTTPS 连接强制启用（HTTP 请求自动跳转 HTTPS）
- 国家标准《信息安全技术 TLS 1.3 安全评估》合规

- 对象存储（OBS）：服务端加密（SSE-AES）
- 密钥管理：使用华为云 KMS（Key Management Service）
- 密钥轮换：自动每 90 天轮换一次

**访问控制**：

**数据脱敏**：
- 所有人工访问数据自动脱敏（手机号、邮箱、身份证隐藏）

### 7.1 内容违规应急

#### 场景 1：用户举报 AI 生成不当内容

**触发条件**：用户举报"AI 生成了违规内容"

**响应流程**：
```
T+0 分 [举报收到]
  → APP 自动记录举报编号
  
T+5 分 [初步评估]
  → 系统自动检查该内容是否仍在线
  → 如仍在线，自动标记"待审核"状态（对其他用户隐藏）
  
T+30 分 [合规员审查]
  → 审核员登录合规后台
  → 查看举报内容+完整对话上下文
  → 判断：违规 / 不违规 / 待商榷
  
T+2 小时 [复审]
  → 审核组长复审初审结果
  → 确认处置决定
  
T+4 小时 [执行处置]
  ├─ 如确实违规 → 删除内容，用户端显示[已被举报的内容]
  ├─ 记录审核意见
  └─ 更新自研词库（如为新敏感内容）
  
T+6 小时 [反馈用户]
  → 邮件/短信通知举报者处置结果
  → 邮件/短信通知内容发布者处置理由
```

#### 场景 2：系统检测到大量同类违规

**触发条件**：自动审核检测到同一模型生成的 10+ 条同类违规内容

**响应流程**：
```
T+0 分 [告警触发]
  → 系统自动告警：[内容风险] 检测到模式化违规
  → 通知合规负责人 + CTO
  
T+30 分 [快速隔离]
  → 暂停该 LLM Provider 的服务（切换到其他 provider）
  → 所有相关内容自动下架
  → 受影响用户自动推送"服务升级中"提示
  
T+2 小时 [原因分析]
  → 技术团队分析：
    ├─ LLM Provider 是否被攻击？
    ├─ Prompt 是否被破解？
    ├─ 是否存在恶意用户输入？
  → 产品团队检查：
    ├─ 是否需要更新 Prompt？
    ├─ 是否需要加强过滤规则？
  
T+4 小时 [制定改进方案]
  → 如 LLM Provider 问题 → 沟通 provider 修复
  → 如 Prompt 问题 → 更新系统指令
  → 如 过滤问题 → 加入新敏感词
  
T+6 小时 [实施改进]
  → 上线改进措施
  → 恢复 LLM Provider 服务
  → 通知用户"AI 服务恢复"
  
T+24 小时 [事后处置]
  → 删除所有违规内容（物理删除）
  → 汇总审核报告
  → 通知网信办（如属重大违规）
```
