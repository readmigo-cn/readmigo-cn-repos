## 三、三道关内容审核流程

### 3.1 第一关：源头白名单

**原理**：预先审核来源，确保源头内容合规

#### Step 1：来源认证

对每个书籍来源（Project Gutenberg、CTEXT 等）进行一次性认证：

| 来源 | 认证方式 | 认证标志 | 更新频率 |
|------|--------|--------|--------|
| Project Gutenberg | 版权符号核实 | ✅ 认证 | 半年一次 |
| Standard Ebooks | 官方许可证核实 | ✅ 认证 | 半年一次 |
| CTEXT | 学术机构背景核实 | ✅ 认证 | 年度一次 |
| Wikisource | 众议规则核实 | ✅ 认证 | 半年一次 |

#### Step 2：初期过滤

对所有来自该源的书籍自动过滤：
- 版权清晰度检查（元数据）
- 文本完整性检查（是否残缺）
- 元数据合规性检查（书名、作者、出版日期）

**标准**：通过率 ≥ 95%（5% 不符合的书籍暂不导入）

### 3.2 第二关：入库自动扫描

**原理**：对所有准备入库的书籍进行自动化和人工审核

#### 自动扫描流程

```
[书籍入库申请]
  ↓
[AI 关键词初筛] (百度 API / 自研词库)
  ├─ 政治敏感词：扫描宪法、国家安全、民族分裂等关键词
  ├─ 色情低俗词：扫描露骨描写
  ├─ 暴力恐怖词：扫描犯罪教程、恐怖内容
  ├─ 宗教极端词：扫描邪教、极端倾向
  └─ 标记敏感章节，形成"敏感章节清单"
  
[输出结果]
  ├─ 绿灯（无敏感词）→ 进入人工审核
  ├─ 黄灯（轻微敏感）→ 重点人工审核
  └─ 红灯（严重敏感）→ 直接拒绝，不进入人工审核
```

**系统选择**：
- **主方案**：百度内容审核 API
- **备选**：阿里云内容安全服务
- **自研**：5,000+ 敏感词库

**扫描覆盖范围**：
- 全文全部字符（包括元数据、目录、前言、正文、后记）
- 特殊符号和特殊字符编码
- 图像（如有扫描版本）

#### 人工审核（初审）

**审核员资格**：
- 本科以上学历
- 通过 40 小时培训（内容审核标准、法律常识、文学素养）
- 通过考核（80 分及格，年度更新）

**审核过程**：

| 阶段 | 时间 | 任务 | 标准 |
|------|------|------|------|
| **敏感章节阅读** | 2-4 小时 | 重点阅读 AI 标记的敏感章节 | 理解上下文含义 |
| **上下文判断** | 1-2 小时 | 判断敏感内容是否违规 | 是否构成违规 |
| **整体评价** | 0.5 小时 | 给出通过/拒绝决定 | 有审核意见记录 |

**审核标准**：见第四章《内容审核工具与机制》

**输出**：
- ✅ **通过**：书籍可入库
- ⚠️ **条件通过**：删除敏感章节后入库，或者标记成人向标签
- ❌ **拒绝**：书籍不入库

### 3.3 第三关：人工复审

**原理**：独立复审初审结果，确保准确性

#### 复审流程

```
[初审意见提交]
  ↓
[审核组长分配] (由资深审核员担任)
  ├─ 通过的书籍：抽查 10%
  ├─ 条件通过的书籍：100% 复审
  └─ 拒绝的书籍：100% 复审
  ↓
[复审评价]
  ├─ 同意初审 → 通过（入库）
  ├─ 不同意初审 → 发回重审
  └─ 疑难案例 → 提交合规负责人最终裁定
  ↓
[最终决定] (入库或不入库)
  ↓
[入库数据库]
  ├─ 通过的书籍：标记 status=approved
  ├─ 拒绝的书籍：标记 status=rejected，记录拒绝原因
  └─ 条件通过：标记相关标签和限制
```

**复审通过率目标**：
- 初审准确度 ≥ 98%（高度一致）
- 因复审改变结果的书籍 ≤ 2%

#### 时限承诺

| 类型 | 审核周期 | SLA |
|------|--------|-----|
| 英文公版（Standard Ebooks 级） | 2-3 天 | 3 天必完 |
| 中文古籍（CTEXT 级） | 3-5 天 | 5 天必完 |
| 其他书籍 | 5-7 天 | 7 天必完 |

**词库规模**：

- 色情低俗词：3,000+ 条
- 暴力恐怖词：1,500+ 条
- 宗教极端词：800+ 条
- 人身攻击词：2,000+ 条

- 精确匹配：完全相同的词汇
- 模糊匹配：谐音、繁简体转换、特殊符号替换

**双重检测**：
```
[内容扫描]
  ↓
[百度 API 检测] → 结果 A
  ↓
[自研词库检测] → 结果 B
  ↓
[结果合并]
  结果 A OR 结果 B = 拒绝
  结果 A AND 结果 B = 高置信拒绝
```

### 4.2 人工审核工作流

#### 审核团队配置

| 角色 | 数量 | 职责 | 资格要求 |
|------|------|------|--------|
| **专职审核员** | 2 名 | 执行初审和日常举报处理 | 本科+、40 小时培训 |
| **审核组长** | 1 名（兼任） | 复审、疑难案例、团队培训 | 本科+、有 1 年审核经验 |
| **合规负责人** | 1 名 | 制定标准、对接监管、最终决策 | 法律或管理背景 |

#### 审核标准操作规程 (SOP)

**初审 SOP**：

```
[领取任务]
  ↓ 审核员从审核队列中领取一本书
  
[元数据检查]
  ├─ 书名是否违规？
  ├─ 作者是否敏感人物？
  ├─ 出版社是否违规？
  └─ 如有违规 → 快速拒绝
  
[AI 敏感章节阅读]
  ├─ 逐个阅读 AI 标记的敏感章节
  ├─ 理解上下文语义
  ├─ 判断是否确实违规
  └─ 记录审核意见
  
[样本阅读]（如 AI 标记较少）
  ├─ 随机选择 3 个章节
  ├─ 通篇阅读 5-10 页
  ├─ 确认 AI 没有遗漏
  └─ 更新审核意见
  
[整体评价]
  ├─ 综合所有信息，给出决定
  ├─ 录入审核系统：通过 / 条件通过 / 拒绝
  ├─ 填写审核意见
  └─ 标记完成
```

**时间分配**（每本书平均 3-5 小时）：
- 元数据检查：5 分钟
- 敏感章节阅读：2-3 小时
- 样本阅读：1-2 小时
- 决策与记录：20 分钟

**复审 SOP**：

```
[接收初审结果]
  ↓ 审核组长从复审队列中领取初审结果
  
[快速查证]（仅针对通过的书籍，抽查 10%）
  ├─ 重新阅读敏感章节
  ├─ 确认初审意见是否准确
  └─ 通过 / 不通过
  
[详细复审]（条件通过和拒绝的书籍 100%）
  ├─ 查看初审意见
  ├─ 重新阅读所有敏感段落
  ├─ 独立判断是否违规
  ├─ 比对初审结果
  └─ 同意 / 驳回 / 要求重审
  
[疑难案例裁定]
  ├─ 初审和复审意见不一致
  ├─ 提交合规负责人裁定
  ├─ 最终决定入库或拒绝
  └─ 记录裁定理由
```
