## 上架前检查清单 (Pre-Launch Checklist)

### 应用准备 (App Readiness)
- [ ] HAP 文件生成完毕，签名证书配置正确
- [ ] App 版本号递增至 1.0.0，Build 号 > 0
- [ ] 应用名称、图标、截图在华为应用市场后台配置完毕
- [ ] 应用简介 (100 字) / 详细描述 (4000 字) 准备完毕
- [ ] 隐私政策链接有效（指向 readmigo.cn ICP 备案页面）
- [ ] 用户协议配置完毕
- [ ] 应用分类、内容分级选择完毕 (3+)

### 合规文件 (Compliance Documents)
- [ ] 软著证书（中国版权中心签发）
- [ ] ICP 备案号（工信部）- 务必在隐私政策第一行展示
- [ ] 营业执照副本（彩色扫描）
- [ ] 法人身份证正反面（彩色扫描）
- [ ] 服务器 IP 白名单确认（华为 CDN）

### 代码审核 (Code Quality)
- [ ] SensorsAnalytics 事件埋点完整（login / register / reading_started / purchase）
- [ ] Sentry 已集成，DSN 正确
- [ ] 崩溃率 < 0.5%（自测版通过 100 次会话无崩溃）
- [ ] 网络请求超时处理完善（5s 超时阈值）
- [ ] 离线功能可用（书籍缓存）
- [ ] 深链接 (deeplink) 正常工作

### 功能测试 (Feature Parity)
- [ ] 登录流程 (手机号 / 微信 / 苹果) - 仅华为登录在 HarmonyOS
- [ ] 首屏推荐流畅（<2s 加载）
- [ ] 搜索功能可用
- [ ] 阅读流畅，无卡顿
- [ ] 朗读功能正常（TTS 集成）
- [ ] 内购流程完整（Pro 订阅）
- [ ] 用户反馈表单可提交

### 性能指标 (Performance)
- [ ] 内存占用 < 150MB (avg)
- [ ] 首屏加载时间 < 2s (3G 网络)
- [ ] 电池消耗在可接受范围内
- [ ] 无过度权限申请

### 文档准备 (Documentation)
- [ ] 华为应用市场提交 SOP (01-huawei-appgallery-submission.md)
- [ ] App Store 中国区提交 SOP (02-ios-appstore-cn-submission.md) [后续]
- [ ] 拒绝应对工具包 (03-rejection-response-toolkit.md)
- [ ] 用户反馈处理 SOP (操作文档)
- [ ] 紧急事件响应 SOP (操作文档)

## 上架提交流程 (Submission Timeline)

### W21 第 1 天：华为应用市场提交
- [ ] 登录华为应用市场开发者后台
- [ ] 上传 HAP、截图、介绍文字
- [ ] 选择应用分类、内容级别
- [ ] 配置权限说明
- [ ] **提交审核**
- [ ] 记录提交时间 + 应用 ID

### W21 第 2-4 天：审核等待期
- [ ] 每日检查审核状态
- [ ] 如有拒绝，立即进入应对流程 (03-rejection-response-toolkit.md)
- [ ] 如需加急，申请加急审核（仅限关键缺陷修复）

### W21 第 5-7 天：审核通过/上架
- [ ] 审核通过后，配置上架时间
- [ ] **上架** (一般在周二至周四)
- [ ] 首日监控到位：DAU / 崩溃率 / 关键错误
- [ ] 启动用户反馈收集

## 上线后 7 天监控清单

### 第 1 天（上架当日）
- [ ] 监控面板在线，实时刷新数据
- [ ] 应用商店链接有效
- [ ] 首 100 用户无崩溃报告
- [ ] API 响应时间 < 200ms
- [ ] 通知推送不重复（去重逻辑）

### 第 2-3 天
- [ ] DAU > 50 (预期)
- [ ] 崩溃率 < 1% (若 > 1% 触发告警)
- [ ] 关键转化率：注册 > 20%
- [ ] Sentry 中无新 issue（自动去重）
- [ ] 用户反馈 < 5 条重复投诉

### 第 4-7 天
- [ ] DAU 趋势分析（增长 / 平台 / 地理分布）
- [ ] 付费转化 > 0（若为 0，检查内购配置）
- [ ] 阅读时长达标（Avg > 5 min/session）
- [ ] 网络错误 < 2%
- [ ] 无新的 P0 issue（Sentry）

### 上线后 SLA
| 指标 | 目标 | 告警阈值 |
|------|------|---------|
| 崩溃率 | < 0.5% | > 1% 红色 |
| API 错误率 | < 1% | > 3% 黄色 |
| 平均响应时间 | < 200ms | > 500ms 黄色 |
| 注册流程完成率 | > 80% | < 60% 红色 |
| 付费转化 | > 5% | < 2% 黄色 |

## 紧急事件处理 (Incident Escalation)

### P0 事件（立即处理）
- 应用完全无法启动
- 所有用户登录失败
- 数据大规模丢失
- **响应时间**: < 15 分钟
- **修复目标**: < 1 小时 + 加急上架

### P1 事件（4 小时内）
- 核心功能不可用（阅读 / 搜索）
- 崩溃率 > 5%
- 付费功能无法使用
- **修复目标**: 2-4 小时

### P2 事件（24 小时内）
- 部分功能异常（排序错误 / 显示问题）
- 用户反馈频繁 (> 3 条同一问题)
- **修复目标**: 1-2 天

---

## 版本回滚预案

如应用上架后 6 小时内发现严重缺陷：
1. 评估影响范围（DAU 占比）
2. 决策：修复 + 加急上架 OR 请求华为下架
3. 若下架，准备通知模板告知用户（见 02-incident-response-sop.md）
4. 修复周期：8 小时内完成测试 + 重新提交
