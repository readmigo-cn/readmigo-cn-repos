## 第一部分：账户与应用配置

### 1.1 App Store Connect 应用创建

1. 登录 https://appstoreconnect.apple.com
2. 点击「我的 App」→「创建应用」
3. 选择应用类型：**应用**（非游戏）
4. 填写基本信息：
   - **应用名称**: 米果智读
   - **主要分类**: 书籍
   - **次要分类**: 阅读
   - **Bundle ID**: com.readmigo.harmony (需与 iOS 项目匹配)
   - **SKU**: readmigo_harmony_2026
5. 保存后系统分配 Apple ID

### 1.2 中国大陆销售区域设置

1. 进入「App」→「定价和销售地区」
2. 勾选「中国大陆」地区
3. 取消勾选其他地区（暂时）
4. 点击「保存」

**重要：** 中国大陆地区需强制实名认证，审核审查更严格

**隐私政策 URL:**
```

```

**App Store 隐私政策检查清单：**

   - ☑️ 电话号码（用于登录）
   - ☑️ 电子邮件地址（备用登录）
   - ☑️ 用户行为数据（阅读历史）
   - ☑️ 诊断数据（崩溃日志）

3. **数据追踪** (App Tracking Transparency)：
   - ❌ 禁用跨应用数据追踪
   - ❌ 不与第三方分享用户追踪信息

## 第六部分：审核与上架

### 6.1 App Store 审核政策核查

在提交前，确认应用符合以下 Apple 政策：

| 政策 | 检查项 | 状态 |
|------|--------|------|
| 应用准确性 | 描述与功能一致，无虚假宣传 | ✅ |
| 元数据 | 截图、图标、关键字真实准确 | ✅ |
| 知识产权 | 未违反第三方版权（书籍已授权） | ✅ |
| 内购政策 | Pro 订阅功能清晰，价格透明 | ✅ |
| 隐私与数据 | 隐私政策透明，数据使用明确 | ✅ |
| 中国法规 | ICP 备案号展示，用户协议完整 | ✅ |
| 内容指南 | 应用内容不违反内容政策 | ✅ |

### 6.2 构建版本准备

1. 在 Xcode 中构建 Archive
   ```
   Xcode → Product → Archive
   ```

2. 上传到 App Store Connect
   ```
   Window → Organizer → Archives
   → 选择最新 Archive → Distribute App
   ```

3. 选择发布选项：
   - 分发方法：App Store
   - 签名证书：选择有效的 Apple Distribution 证书
   - Provisioning Profile：选择对应的 Profile

### 6.3 提交审核

1. 在 App Store Connect 中，进入「App 版本」
2. 确认所有必填项已完成：
   - ✅ 应用图标（1024×1024px）
   - ✅ 应用截图（至少 2 张）
   - ✅ 应用预览（可选）
   - ✅ 应用名称与副标题
   - ✅ 应用描述、关键字
   - ✅ 隐私政策 URL
   - ✅ 用户协议 URL
   - ✅ 支持 URL（support@readmigo.cn）
   - ✅ 内容分级
   - ✅ 建构版本（已上传）

3. 点击「提交以供审核」
4. 回答 App Store 相关问题：
   - **此 App 是否使用 IDFA？** → 否（除非涉及广告）
   - **此 App 是否包含加密？** → 是（HTTPS 传输）
   - **此 App 是否由美国出口？** → 是
   - **此 App 是否有医疗声明？** → 否

5. 确认并提交

**提交成功后记录：**
```
提交时间：2026-05-0X XX:XX
预计审核完成：2026-05-0X（24-48 小时）
审核单号：[显示的 ID]
```

## 第七部分：中国特色审核考察点

### 7.1 ICP 备案号显示

**必须在应用内显示 ICP 备案号：**

1. 在「关于」→「法律信息」或「隐私政策」顶部显示
   ```
   京 ICP 备 [XXXXXXX] 号
   ```

2. ICP 备案号链接至工信部备案查询系统（可选但推荐）
   ```
   https://beian.miit.gov.cn/
   ```

3. Apple 审核人员会验证备案号真实性

### 7.2 内容审查敏感词

避免以下内容（容易被 App Store 中国区驳回）：

- ❌ 涉及政治、宗教、民族的书籍宣传
- ❌ 涉及医疗、心理治疗的声称（除非专业资质）
- ❌ 涉及赌博、色情、暴力的书籍
- ❌ "破解"、"免费获取付费内容" 等诱导语
- ❌ 过度的数据追踪声明

### 7.3 国内支付方式

**App Store 支付在中国的配置：**

- Apple 支持支付宝、微信支付（通过 Apple 代理）
- 用户在中国 App Store 购买时，使用支付宝 / 微信完成支付
- 确保 In-App Purchase 配置与 HarmonyOS 版本保持一致

## 第八部分：常见拒绝原因

### 8.1 中国区常见驳回原因

| 原因代码 | 具体原因 | 修复方案 |
|---------|---------|--------|
| 4.3 | ICP 备案号不完整 | 在应用 → 关于 中添加完整 ICP 号 |
| 2.1 | 应用描述与功能不符 | 更新描述文案，确保实现了所有声称的功能 |
| 3.1 | 隐私政策不完整 | 添加个人信息保护内容 |
| 4.1 | 内购价格不透明 | 确保 Pro 订阅价格在应用内明确显示 |
| 5.2 | 应用包含虚假内容 | 检查截图、描述是否有虚假宣传 |

### 8.2 应对流程

1. **收到拒绝后**：立即保存拒绝信息截屏
2. **分析根因**：对照上表，确定修复方案
3. **修复内容**：
   - 修改代码（若涉及功能）
   - 更新文案、截图、隐私政策链接
4. **本地验证**：确保修复内容有效
5. **撰写开发者回复**：
   ```
   尊敬的 Apple 审核团队，

   感谢您的审核反馈。我们已根据您的意见进行以下修改：

   1. 在应用「关于」页面补充了完整的 ICP 备案号（京 ICP 备 XXXXXXX 号）
   2. 更新了应用描述，确保功能说明与实现保持一致
   3. 验证了隐私政策链接的有效性

   我们已上传新的构建版本 (v1.0.0, Build 2)，请重新审核。

   谢谢！
   米果智读团队
   support@readmigo.cn
   ```
6. **重新提交**：点击「重新提交以供审核」
